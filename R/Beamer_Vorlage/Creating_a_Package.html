Configure Build Tools

The package shell was not built for roxygen2. As such, we need to make some manual adjustments to the files before we start.

  • Delete the NAMESPACE file
  • Delete the example function Hello.R (you could also just overwrite this with your own functions, but we are going to get rid of it) *** ## Roxygenise

Now that we have gotten rid of these files, we need to replace them with roxygen2 versions

We can do this by using a function from devtools: devtools::document()
The document function calls roxggen2::roxygenise() and creates roxygen versions of your code. If we run it now, it will create a new NAMESPCE file and also a LICENSE file (we will come back to licensing shortly). *** ## Add functions

Lets take a very simple function:

times_2 <- function(x){
  return(x*2)
}

At the moment, R knows that this is a function, but not that this function should be part of our package. For this we need to add some roxygen2 syntax. #' Note that there is a spaceincluded here. #' @param and #' @return both need to start with a capital letter and end with punctuation, or you will get a warning when you run devtools::check()

#' What does this function do?
#' 
#' @param x What is x?
#' @return What value do you get returned to you?
#' @examples times_two(56)
#' @export


times_two <- function(x){
  return(x*2)
}

If we run devtools::document() again, our function should be roxygenised, and an times_two.Rd file in the man folder will be created.

DESCRIPTION FILE

The DESCRIPTION file is the file which stores all the important meta data about your package, including information about the author/ creator and any dependencies your package needs in order to run the functions you have provided. The fields provided in the shell include:

Package: ShrekNums
Type: Package
Title: What the Package Does (Title Case)
Version: 0.1.0
Author: Who wrote it
Maintainer: The package maintainer <yourself@somewhere.net>
Description: More about what it does (maybe more than one line)
    Use four spaces when indenting paragraphs within the Description.
License: 
Encoding: UTF-8
LazyData: true
RoxygenNote: 7.1.0

Lets fill the description file out. the Package:, Type:, Version:, Encoding:, LazyData: and RoxygenNote: fields can remain as they are for now. We will also deal the License: field in the next section.

Title:

This title is what shows up at the top when someone searches the help function for your package. Make it short and informative.

Author:

Self explanitory. If there are more than 1 authors, contributors or creators, you can change the syntax a bit:

Authors@R: c(
    person(given = "SALLY",
           family = "PERSON",
           role = c("aut", "cre"),
           email = "sally.person@uni.ac.at"),
    person(given = "BOB",
           family = "OTHER",
           role = "ctb"))

Description:

This is a longer description of what your package does and why it is useful. It also appears in the help section.

License:

Now that we have some content to our package, we need to think about licensing it. a list of different license options can be found here. You can either copy one of the licenses into a .txt file and name it LICENSE, or use a function from the usethis package.

usethis::use_mit_license("YOUR NAME")
usethis::use_gpl3_license("YOUR NAME")
usethis::use_agpl3_license("YOUR NAME")
usethis::use_lgpl_license("YOUR NAME")
usethis::use_apl2_license("YOUR NAME")
usethis::use_cc0_license("YOUR NAME")
usethis::use_ccby_license("YOUR NAME")

Using one of these functions creates a LICENSE.md file. You need to keep the LICENSE.txt file or you will get an error message when you do your checks.

you will also need to add the following line to the description file:

License: MIT + LICENSE file
